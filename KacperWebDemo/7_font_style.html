Notice the patterned behavior that can be subdivided • factorial(n) = n! = n * factorial(n - 1) • factorial(3) = 3! = 3 * factorial(2) = 3 * 2 * factorial(1) = 3 * 2 * 1 = 6 • This means the base case is when n = 1. (Or if we don’t want to error handle, n < 2). Notice that this can be made to be a boolean named constant, and even the stopping point could be as well. • The update comes from the number decreasing as it is passed into future factorial calls (recursive step).
